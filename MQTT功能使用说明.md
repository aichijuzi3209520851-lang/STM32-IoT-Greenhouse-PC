# DHT11上位机 - MQTT功能使用说明

## 功能概述

DHT11温湿度上位机现已集成MQTT功能，可以将串口采集到的温湿度数据实时发送到本地或远程的MQTT服务器。这使得数据可以被其他设备订阅和接收，实现物联网数据共享。

## 主要特性

### 1. MQTT客户端连接
- 支持连接到本地或局域网内的MQTT服务器
- 可自定义MQTT服务器地址和端口
- 实时显示连接状态
- 支持优雅的断开连接

### 2. 自动数据发布
- 数据解析成功后自动发布到MQTT服务器
- 可通过复选框开启/关闭自动发布
- 数据以JSON格式发布，便于解析

### 3. 灵活的配置
- 可随时修改服务器地址
- 支持不同局域网环境下的快速切换
- 连接状态实时反馈

## 使用方法

### 步骤1: 启动MQTT服务器

本项目基于EMQX MQTT服务器，启动方法：

1. **进入EMQX目录**
   ```
   cd emqx-5.0.26-windows-amd64\bin
   ```

2. **启动服务器**
   ```
   .\emqx.cmd start
   ```

3. **验证服务器状态**
   - 打开浏览器访问: `http://localhost:18083`
   - 默认用户名: `admin`
   - 默认密码: `public`

4. **停止服务器**（需要时）
   ```
   .\emqx.cmd stop
   ```

### 步骤2: 配置上位机MQTT连接

1. **打开DHT11上位机程序**

2. **在控制面板右侧找到MQTT配置区域**
   - "MQTT服务器"输入框
   - "连接MQTT"按钮
   - "断开MQTT"按钮
   - 连接状态指示器
   - "自动发布数据到MQTT"复选框

3. **输入MQTT服务器地址**
   - 默认地址: `192.168.184.221:1883`
   - 格式: `IP地址:端口`
   - 本地服务器: `localhost:1883` 或 `127.0.0.1:1883`
   - 局域网服务器: 输入局域网IP和端口（如 `192.168.1.100:1883`）

### 步骤3: 连接MQTT服务器

1. **点击"连接MQTT"按钮**
   - 状态指示器显示"连接中..."（橙色）
   - 接收日志显示连接进度

2. **连接成功**
   - 状态指示器显示"已连接"（绿色）
   - "连接MQTT"按钮变为不可用
   - "断开MQTT"按钮变为可用
   - 服务器地址输入框锁定
   - 接收日志显示连接成功信息

3. **连接失败**
   - 状态指示器显示"连接失败"（红色）
   - 弹出错误提示框
   - 接收日志显示错误详情
   - 检查服务器地址、网络连接和服务器状态

### 步骤4: 数据发布

1. **连接串口并接收数据**
   - 按照原有流程连接串口
   - 选择正确的数据格式
   - 串口开始接收温湿度数据

2. **自动发布数据**
   - 确保"自动发布数据到MQTT"复选框已勾选
   - 每当接收到数据并解析成功时，会自动发布到MQTT
   - 接收日志显示: `[MQTT发送] 温度:XX°C 湿度:XX%`（蓝色）

3. **手动控制发布**
   - 取消"自动发布数据到MQTT"复选框：停止发布
   - 勾选"自动发布数据到MQTT"复选框：继续发布

### 步骤5: 断开连接

1. **点击"断开MQTT"按钮**
   - 状态指示器显示"未连接"（灰色）
   - "连接MQTT"按钮恢复可用
   - "断开MQTT"按钮变为不可用
   - 服务器地址输入框解锁

2. **程序关闭时自动断开**
   - 关闭程序时会自动断开MQTT连接
   - 无需手动操作

## 数据格式说明

### 发布主题
- **默认主题**: `DHT11/data`
- 其他设备可订阅此主题接收温湿度数据

### 数据JSON格式
```json
{
    "timestamp": "2025-12-18 15:30:45.123",
    "temperature": 25.5,
    "humidity": 60.2,
    "deviceId": "DHT11_Sensor_01",
    "dataFormat": "格式1: Temperature:XX Humidity:XX"
}
```

### 字段说明
- **timestamp**: 数据采集时间戳（精确到毫秒）
- **temperature**: 温度值（摄氏度）
- **humidity**: 湿度值（百分比）
- **deviceId**: 设备标识符（固定为DHT11_Sensor_01）
- **dataFormat**: 当前使用的数据解析格式

## 在局域网中使用

### 场景1: 本地测试
```
MQTT服务器地址: localhost:1883
或: 127.0.0.1:1883
```

### 场景2: 同一局域网内
```
1. 在服务器电脑上查看IP地址:
   - 打开cmd
   - 输入: ipconfig
   - 记录IPv4地址,如: 192.168.1.100

2. 在上位机中输入:
   MQTT服务器地址: 192.168.1.100:1883
```

### 场景3: 切换到其他电脑的服务器
```
1. 确保两台电脑在同一局域网
2. 获取服务器电脑的IP地址
3. 修改上位机的MQTT服务器地址
4. 点击连接即可
```

## 常见问题

### Q1: 连接MQTT失败怎么办？
**A:** 请检查：
1. MQTT服务器是否已启动（访问 http://localhost:18083 确认）
2. 服务器地址和端口是否正确
3. 防火墙是否阻止了连接
4. 如果是局域网连接，检查两台电脑是否在同一网络

### Q2: 数据没有发送到MQTT怎么办？
**A:** 请确认：
1. MQTT已成功连接（状态显示为"已连接"）
2. "自动发布数据到MQTT"复选框已勾选
3. 串口数据解析成功（接收日志显示解析成功）
4. 查看接收日志是否有MQTT发送记录

### Q3: 如何查看MQTT接收到的数据？
**A:** 有以下几种方法：
1. 使用EMQX Dashboard
   - 访问 http://localhost:18083
   - 登录后进入"WebSocket"或"监控"页面
   - 订阅`DHT11/data`主题查看数据

2. 使用MQTT客户端工具（如MQTTX）
   - 连接到MQTT服务器
   - 订阅`DHT11/data`主题
   - 实时查看接收到的JSON数据

3. 编写程序订阅主题
   - 使用Python、Node.js等编写MQTT客户端
   - 订阅并处理数据

### Q4: 可以同时连接多个上位机吗？
**A:** 可以。每个上位机会生成唯一的客户端ID，不会冲突。所有上位机都可以向同一主题发布数据。

### Q5: 端口18083和1883有什么区别？
**A:**
- **1883端口**: MQTT协议端口，上位机连接使用
- **18083端口**: EMQX Dashboard Web管理界面端口，浏览器访问使用

## 技术说明

### MQTT协议
- **QoS级别**: AtLeastOnce (QoS 1)
- **Retain标志**: false（不保留消息）
- **Clean Session**: true（清除会话）

### 客户端ID
- 格式: `DHT11_Client_[GUID]`
- 每次连接生成唯一ID
- 避免客户端冲突

### 数据发布频率
- 根据串口数据接收频率实时发布
- 无额外延迟
- 不影响串口数据接收和UI更新

## 注意事项

1. **网络安全**: 本地MQTT服务器默认无认证，仅适用于内网环境
2. **性能影响**: MQTT功能对原有功能无影响，可独立开关
3. **数据隐私**: 局域网内的设备都可以接收到发布的数据
4. **服务器状态**: 确保MQTT服务器持续运行，否则连接会断开
